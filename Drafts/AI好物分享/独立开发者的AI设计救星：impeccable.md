你让 AI 帮你做了个界面，出来的东西大概长这样：

- 标题用 Inter，正文也用 Inter，按钮还是 Inter
- 主色调是蓝紫渐变，配上白色描边的玻璃拟态卡片
- 卡片里面套卡片，卡片里面还有卡片

你改了一版，换了个颜色，结果还是同款气质。换个提示词，依然如此。

这不是你的问题，也不是 AI 的 bug。

---

## 为什么 AI 画出来的界面都长一个样

所有大模型都从同一批数据训练出来的——那批数据里充满了 Bootstrap 模板、Tailwind UI 示例、Dribbble 热门作品。AI 学会的是"什么设计看起来像设计"，而不是"什么设计适合这个产品"。

没有干预，AI 就会回归最安全的平均值。

impeccable 把这些典型反模式整理成了 Gallery of Shame：

- **Inter Everywhere** — 没有理由地把 Inter 用遍全站
- **Cardocalypse** — 无止境地嵌套卡片
- **Purple Gradients** — 默认紫蓝渐变当主视觉
- **Side-Tab Cards** — 左侧粗色边框当成"有设计感"的捷径
- **Template Layouts** — 所有页面都像从同一个模板克隆的
- **Bad Contrast** — 文字对比度不够，可读性差

认出来了吗？这就是你一直在和 AI 较劲的那些东西。

---

## impeccable 是什么

impeccable 是一个开源工具包（Apache 2.0），专门解决这个问题。它做了两件事：

1. 给 AI 装了一套系统性的设计知识库——字体学、色彩理论、空间设计、交互设计、动效……让 AI 真的理解这些原则，而不只是模仿模板
2. 提供 21 个精准的设计指令，让你能用具体的语言指挥 AI 往特定方向改

不需要你懂设计，你只需要知道现在哪里有问题，挑对应的指令就行。

支持 Claude Code、Cursor、Gemini CLI、VS Code Copilot 等 9 个平台。

## 怎么安装

Claude Code 用户两选一：

```bash
# 推荐，自动检测工具
npx skills add pbakaus/impeccable
```

或者直接在 Claude Code 里：

```
/plugin marketplace add pbakaus/impeccable
```

装完之后，先跑一次 `/impeccable teach`，把设计知识加载进 AI 的上下文。只需要做一次，之后所有对话都带着这套认知。

## 21 个 skill 完全解析

按使用场景分成五组，逐一看清楚每个 skill 干什么。

### 第一组：动手之前先规划

**`/shape`**
在写代码前，先做 UX 和 UI 的规划。它会和你进行一次结构化的设计探讨，产出一份设计简报，包含交互策略、视觉方向、约束条件，再交给 AI 去实现。

适合用在：功能比较复杂、不确定从哪里下手、不想写完再推翻的时候。

**`/impeccable`**
主命令。直接让 AI 生成高质量、有设计感的界面，主动绕开 Gallery of Shame 里的那些反模式。也可以用 `/impeccable craft` 走完整的"先规划后实现"流程，或者 `/impeccable teach` 单独加载设计上下文。

---

### 第二组：检查和诊断

**`/audit`**
技术质量扫描。跑完会给你一份评分报告，覆盖无障碍访问、性能、主题一致性、响应式适配、以及反模式检测，每个问题标注 P0–P3 优先级和修复建议。

不知道从哪里下手？先跑 `/audit`，它会告诉你。

**`/critique`**
UX 视角评审。不只看技术问题，而是从信息架构、视觉层级、认知负担、情感共鸣等维度打分，还会模拟不同用户画像来发现潜在问题。

`/audit` 查技术，`/critique` 查体验，两个维度互补。

---

### 第三组：视觉调整

**`/typeset`**
修字体问题：字体选择、层级关系、字号大小、字重、行距。让文字看起来是有意图地被排版的，而不是随手堆上去的。

**`/colorize`**
当界面太灰、太单调，用这个加上策略性的颜色，让视觉更有层次感和表达力。

**`/arrange`**
修布局和间距：网格单调、间距不统一、视觉流动感差、对齐有问题——这些都是它的目标。

**`/bolder`**
界面太保守、太无聊、没有个性？用这个推一把，在保持可用性的前提下增加视觉冲击力和性格。

**`/quieter`**
反过来：界面太抢眼、太吵、视觉噪音太多？用这个降噪，让设计更克制、更精致。

**`/distill`**
比 `/quieter` 更彻底。去掉所有不必要的元素，把界面剥到最核心的东西。追求极简风格时用。

---

### 第四组：体验和细节

**`/animate`**
为界面加上有意义的动效和微交互——hover 效果、过渡动画、状态变化反馈。让界面感觉活着，而不是静态的。

**`/delight`**
加惊喜感。那些让用户会心一笑的小细节：意想不到的动画、个性化的空状态插画、有趣的交互反馈。把"能用"提升到"好玩"。

**`/onboard`**
专门优化新用户体验：首次进入的引导流程、空状态的设计、快速让用户看到价值的路径。留存从这里开始。

**`/clarify`**
改文案。按钮上写"提交"还是"立即开始"？错误提示是"操作失败"还是"网络不稳定，请稍后重试"？这个 skill 处理所有 UX 文字，让界面更好理解。

---

### 第五组：打磨和生产化

**`/polish`**
上线前的最后一遍检查。对齐、间距、视觉一致性、微小细节……把"做完"打磨成"做好"。

**`/harden`**
让界面经得起真实使用：错误状态设计、长文本溢出处理、国际化支持、边缘情况覆盖。Demo 看起来好看不够，用户数据来了还得撑得住。

**`/optimize`**
UI 性能诊断：加载速度、渲染效率、动画流畅度、图片优化、bundle 大小。感觉慢或者卡，跑这个。

**`/adapt`**
响应式适配：断点设置、流式布局、触摸目标大小、跨设备兼容。移动端、桌面端都要好用。

**`/normalize`**
当多人协作或者长期迭代之后，界面各处的间距、颜色、组件开始"漂移"不一致了——这个 skill 把一切拉回设计系统的规范。

**`/extract`**
把重复的 UI 模式提炼成可复用的组件和设计 token，沉淀进设计系统。项目规模变大之后必然用到。

**`/overdrive`**
特殊武器。要做出让人眼前一亮的效果：着色器、弹性物理动画、滚动驱动的视差、60fps 的流畅体验。做落地页、产品主页、想出圈的时候用。

## 实战：怎么真正用好它

光知道 21 个 skill 是什么还不够，关键是什么时候用哪个、怎么组合。

### 从零开始做一个新界面

```
/shape           → 先做设计规划，明确方向
/impeccable      → 生成初版界面
/audit           → 扫描技术问题
/critique        → 检查用户体验
/typeset         → 修字体
/arrange         → 修布局
/colorize        → 如果颜色太单调
/polish          → 上线前最终打磨
/harden          → 确保边缘情况都覆盖了
```

### 已有界面，感觉哪里不对

先用 `/audit` + `/critique` 定位问题，再根据报告选对应的 skill。不要一上来就随便跑 `/polish`，那只是掩盖问题，不是解决问题。

### 界面"没问题"但就是不好看

跑 `/critique` 听听 AI 的 UX 诊断，再根据症状选 skill：
- 视觉太平淡 → `/bolder`
- 太乱太吵 → `/quieter` 或 `/distill`
- 配色无聊 → `/colorize`
- 字体没层次 → `/typeset`

### 准备上线的最后一公里

```
/harden   → 处理所有边缘情况
/optimize → 性能检查
/adapt    → 移动端再过一遍
/polish   → 最终收尾
```

---

## 一个容易踩的坑

不要每次都跑 `/polish`。

`/polish` 是收尾 skill，它假设你的问题已经解决了，只差最后对齐和细节。如果根本问题是布局混乱或者字体层级不对，`/polish` 修不了——你需要先跑 `/arrange` 或 `/typeset`。

用对顺序：诊断 → 定向修复 → 最后收尾。
