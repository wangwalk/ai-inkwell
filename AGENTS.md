# AI Inkwell — Walker 的内容创作工作台

## 你在和谁协作

Walker，技术内容创作者。四个专栏：
- **Codex 实战指南** — 深度教程，面向开发者和效率爱好者
- **AI 自动化食谱** — 具体场景的自动化方案，面向所有人
- **被忽略的 AI 工具** — 小众但好用的工具推荐
- **本周 AI 圈** — 周更资讯，筛选+点评

## 写作风格

- 朋友式对话，不是专家讲课
- 真诚无私，目的是让读者觉得"这对我有用"
- 干货优先，不灌水不凑字数
- 说人话，拒绝"赋能""抓手""打法"等黑话
- 有观点，敢说好坏，不做中立复读机
- 段落短，适配手机阅读

## 写作原则

1. 开头直接抛出最有价值的信息，不铺垫不寒暄
2. 正文围绕"读者能拿走什么"组织，不围绕"我知道什么"
3. 代码/配置要能直接复制使用，不写伪代码
4. 结尾不总结复述，留一个值得思考的点或下一步行动
5. 配图服务于理解，不是装饰

## Vault 结构

```
00-Inbox/          # 随手记的想法、链接、素材，零门槛进入
01-Drafts/         # 按专栏分目录，写作中的内容
02-Published/      # 发布后移入，frontmatter 记录各平台链接
03-Resources/      # 参考资料、竞品分析、数据源
04-Learning/       # 学习笔记、面试题整理，非正式创作
templates/         # 写作模板
```

图片跟随文章放在同目录的 `imgs/` 子目录下，不单独集中存放。

## 发布平台

按优先级：公众号 > 知乎 > 小红书 > X > 个人博客

各平台适配要点：
- **公众号/知乎/博客**：完整长文
- **小红书**：提炼核心观点，图文为主，1000字内，代码放图或省略
- **X**：一条精华 + 关键截图，或 thread 展开

## 工作流

1. 素材丢进 `00-Inbox/`
2. 开写时从模板创建，放入 `01-Drafts/{专栏名}/`
3. 写完审阅后更新 status: ready
4. 发布到各平台，把链接填入 frontmatter
5. 移入 `02-Published/`
